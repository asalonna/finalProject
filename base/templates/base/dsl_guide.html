DSL Guide


Introduction

This DSL is designed to assist the creation of programming questions, by allowing generation of a range of questions allowing the use of random values for each student, and their corresponding expected answers.

This DSL also allows implementation of restrictions in the code. With this DSL, you can make it so that students have to use a variety of different concepts to answer your questions.

The DSL also includes answer checking. This helps relieve the burden of having to spend a lot of marking revision questions in a large class, as well as forces students to have to use specific concepts in their answers (such as an if statement).


Question Creation

To define a question, you should start with the keyword “Question” and Content must be defined between “<>” tags.

There must be a space between “Question” and the start of the tags.

For example, a simple Hello World would be written as follows:

Question <”Print out the string ‘Hello World!’”>

Text

Text is written as anything between two speech marks and can contain any non-speech mark character.

Random Integers

Random integers can be specified by typing %randInt and then a variable name. It must also end with %:

%randInt random_number%

When generating a random integer, you must also declare the range in which you want the random number to occur, following this format:

%randInt random_number = x,y%.

x is the start of the range and y is the limit. x and y must be integers.

An example question using this is:

Question <"Print the number " %randInt x = 1,100% ". (HINT: Use ‘System.out.println();’)">

Answer <x>

Random Float

Same as random integers, except you write %randFloat random_float% for example.

Random Order

Random Order allows you to randomise the order of elements in a given array.

It is declared by writing #{items here}#.

The items in the array must be separated by a comma.

Example: #{North, East, South, West}#


Answer Checking

Answer checking in the DSL works using expected output matching.

Answers must be declared with the keyword “Answer” then followed by “<>” with content between the tags.

The answer to the hello world question above would be written:

Answer <”Hello World!”>

Text

Answer text must be declared within speech marks (“”).

Script

An Answer can also be the output of a script.

Scripts are declared as follows:

python { {15*37} }

Functions

There are two built-in functions that users can utilise.

The first is Order:

This function makes the expected answer an ordered array. This can also be used with randomly generated numbers.

Example:

Question <”Print out the numbers “ %randInt a = 10,20% “, “ %randInt b = 1,10% “,” %randInt c = 15,25% “,” %randInt d = 1,5% “,” %randInt e = 1,30% “ in ascending order, separated by commas.”>

Answer <order{a,b,c,d,e}>

The second is Sequence:

Can create answers involving a start, step, and an end.

Example:

Question <"Print the numbers " %randInt x = 1,5% " up to (but not including) " %randInt y = 10,20% ", separated by commas. This question requires the use of 1 for loop. (HINT: Use System.out.print())">

Answer <sequence{from x until y step 1}>

Variables

Example: You can generate a number and store it in a variable, say x. You can reference x as the Answer too.

Question <"Print the number " %randInt x = 1,100% ". (HINT: Use ‘System.out.println();’)">

Answer <x>

Repetitions

Can specify how many times something should be done. Example: how many times a string should be printed. This can also use the randomly generated integers.

Example:

Question <"Print the string ‘I love Java!’ " %randInt x = 4,9% " times, each on the same line with no delimiter. This question requires the use of 1 for loop.">

Answer <"I love Java!"*x>


Restrictions

Like Question and Answer, Restriction is declared:

Restrict <content>

When declaring multiple restrictions, it is recommended to include a space between them for readability.

You can declare different types of restrictions for the same question, such as both an if and a for loop:

Restrict <for if>

Note: This does not take into account the order in which the restrictions are written. For the above example, the DSL will consider an answer to be correct if it uses at least one for loop and one if statement, and the order will not cause a submission to be marked as wrong.

If

You can specify how many times an If statement should be used.

Restrict <if>

Multiple ifs:

Restrict <if if if>

For

You can specify how many times a For loop should be used.

Restrict <for>

Multiple for loops:

Restrict <for for>

While

You can specify how many times a While loop should be used.

Restrict <while>

Multiple while loops:

Restrict <while while>


Commenting

A line of this DSL can be commented out using //.
